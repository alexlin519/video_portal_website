# 测试备份文件说明

这里提供了几个测试用的备份文件，用于测试不同的导入场景：

## 1. `test_backup_template.json` - 标准测试模板
包含常见的测试数据：
- **收藏项**: [1, 2, 9, 28]
- **置顶项**: [1, 11, 27, 28]
- **自定义排序**: 所有分类都有自定义排序
- **用户添加项**: 在 `hehe-两性`、`新游试玩`、`daily-random` 中添加了测试项
- **删除项**: [11, 20]
- **筛选设置**: 包含类别、子类别和排除项
- **侧边栏**: 展开状态

## 2. `test_backup_empty.json` - 空数据模板
用于测试清空所有设置：
- 所有数据都为空或默认值
- 可以用来重置所有用户设置

## 3. `test_backup_max_data.json` - 最大数据模板
包含大量测试数据：
- **收藏项**: 11 个项目
- **置顶项**: 11 个项目
- **用户添加项**: 6 个不同位置的测试项
- **删除项**: 3 个项目
- **筛选设置**: 包含多个类别和排除项
- **侧边栏**: 折叠状态
- **多个子类别排序**: 3 个分类都有子类别排序

## 使用方式

1. 在网页中点击"📤 导入"按钮
2. 选择对应的测试文件
3. 确认导入后，页面会自动刷新并应用设置

## 测试场景建议

- **标准模板**: 测试正常的导入/导出流程
- **空数据模板**: 测试清空设置功能
- **最大数据模板**: 测试边界情况和大量数据处理

## 注意事项

- 导入前建议先导出当前设置作为备份
- 导入后会覆盖所有现有设置
- 某些 ID 可能在你的 `data.json` 中不存在，这是正常的（用于测试错误处理）
